# Jumma Time

Веб-платформа для мечетей: пятничные хутбы, сунны пятницы и другие материалы на трёх языках — **казахском, русском и английском**.

Миссия — помочь прихожанам лучше понимать и сохранять содержание пятничной хутбы (особенно тем, кто не владеет казахским на достаточном уровне), и дать имамам простой инструмент для публикации материалов.

---

## Статус проекта: MVP готов

Это первая рабочая версия (MVP), реализующая весь функционал, описанный в исходном концепте продукта. Проект **не запущен в продакшен** — перед реальным использованием требуется:

- наполнение реальным, проверенным имамами контентом (сид-данные — тестовые примеры);
- смена паролей демо-админов и `AUTH_SECRET`;
- перенос базы данных с локального SQLite на хостed БД (см. [«Деплой»](#деплой));
- шариатская проверка формулировок (это прямо обозначено как риск в исходном концепте).

Функционально всё, что описано в разделе «MVP-функционал» концепта, реализовано и протестировано вручную в браузере (десктоп-вёрстка; публичные страницы, сохранение хутб, вход в админку, создание/редактирование/удаление хутб и материалов).

---

## Технологический стек

Проект — **full-stack Next.js-монолит** (фронтенд и бэкенд в одном приложении, один деплой), а не связка «отдельный React SPA + отдельный Express API», как было в исходном концепте. Это решение осознанно выбрано вместо изначально предложенного `React + Node/Express + MongoDB`, так как для MVP с одним разработчиком это быстрее в разработке, проще в деплое и не требует поднимать/поддерживать два сервиса и CORS.

| Область | Технология | Зачем |
|---|---|---|
| Фреймворк | **Next.js 16** (App Router, React 19, TypeScript) | Один проект для фронта и бэка, SSR из коробки, файловая маршрутизация |
| Стили / UI-кит | **Tailwind CSS v4** + **shadcn/ui** (на базе [Base UI](https://base-ui.com), не Radix) | Быстрая сборка интерфейса, доступные компоненты, легко кастомизировать под фирменную палитру |
| Локализация | **next-intl** | Маршруты `/kk`, `/ru`, `/en`, серверные и клиентские переводы, переключатель языка |
| База данных | **SQLite** (файл `dev.db`) через **Prisma 7** + **`@prisma/adapter-libsql`** | Prisma 7 требует driver adapter вместо старого движка; libSQL выбран вместо `better-sqlite3`, потому что последний требует нативную компиляцию (node-gyp/Visual Studio), которой нет на машинах без C++ тулчейна. Схема не привязана к SQLite — замена на Postgres/Turso — это смена одной строки `DATABASE_URL` и адаптера |
| Авторизация | **NextAuth (Auth.js) v5**, Credentials-провайдер, JWT-сессии, пароли — `bcryptjs` | Простой email/пароль для имама, без внешних OAuth-провайдеров |
| Формы/валидация | **Zod** + React Server Actions (`useActionState`) | Валидация на сервере без отдельного REST-слоя |
| QR-код | **`qrcode`** | Генерация QR на лету на сервере (страница `/qr`) |
| Хостинг (рекомендуется) | **Vercel** (фронт+бэк) + управляемая Postgres/Turso БД | См. раздел «Деплой» |

### Особенности версий (не как в обучающих данных большинства LLM)

Часть используемых библиотек — свежие мажорные версии с ломающими изменениями по сравнению с тем, что типично «знают» языковые модели. При работе с кодом это стоит держать в голове (см. также `AGENTS.md`):

- **Next.js 16**: файл `middleware.ts` переименован в `src/proxy.ts` — это тот же обработчик (next-intl), просто новая file convention. Экспорт остался default-функцией.
- **Base UI** (`@base-ui/react`, на нём собран `shadcn/ui` в этом проекте) — это **не Radix UI**. У `Menu.Item` нет пропса `onSelect`, только `onClick` (в Radix — наоборот). Если код выглядит «как обычно для shadcn», но клик по пункту меню ничего не делает — велика вероятность именно этой путаницы.
- **Prisma 7**: нужен driver adapter (`@prisma/adapter-libsql` для SQLite/Turso), «голый» `DATABASE_URL` без адаптера не заработает.

## Функционал

### Публичная часть (доступна без входа)

Все страницы локализованы (`/kk/...`, `/ru/...`, `/en/...`), с переключателем языка в шапке.

**Мультимечетность.** Платформа обслуживает сразу несколько мечетей на одном деплое. В шапке рядом с языком — переключатель мечети (виден только если мечетей больше одной); выбор запоминается в cookie `mosque` на год. Хутбы и материалы разделов (сунны, рекомендации, напоминания, достоинства пятницы, вводный блок Аль-Кахф) принадлежат конкретной мечети и видны только когда она выбрана — прямая ссылка на хутбу чужой мечети отдаёт 404. Сами 110 аятов Аль-Кахф — общие для всех мечетей (это текст Корана, а не контент конкретной общины).

| Страница | Маршрут | Описание |
|---|---|---|
| Главная | `/{locale}` | Приветствие, кнопки «Смотреть хутбы» / «QR-код мечети», хутба недели (текущей мечети), карточки-разделы |
| Каталог хутб | `/{locale}/khutbahs` | Список опубликованных хутб (все периоды), поиск по названию/дате |
| Хутба | `/{locale}/khutbahs/[slug]` | Полный текст на выбранном языке, кнопка «Сохранить» (в `localStorage`, без регистрации пользователя), кнопка «Поделиться» (Web Share API / копирование ссылки) |
| Сохранённые | `/{locale}/saved` | Список хутб, сохранённых на этом устройстве |
| Сунны пятницы | `/{locale}/sunnah` | Список рекомендованных действий из сунны |
| Рекомендуемые действия | `/{locale}/recommended-actions` | Что стоит сделать в пятницу |
| Напоминания | `/{locale}/reminders` | Важные напоминания (например, час принятия дуа) |
| Сура Аль-Кахф | `/{locale}/al-kahf` | Вводный текст о достоинстве чтения суры + **полный текст всех 110 аятов** прямо на странице: арабский оригинал (издание Усмани), транслитерация и перевод. Внизу — дополнительная ссылка на Quran.com |
| Достоинства пятницы | `/{locale}/friday-virtues` | Материалы о статусе пятницы в исламе |
| QR-код мечети | `/{locale}/qr` | QR-код на главную страницу — для печати на стенде мечети |

**Важно про «Сохранённое»:** по концепту не предполагалась полноценная регистрация обычных пользователей, поэтому сохранение хутб реализовано через `localStorage` браузера (без БД, без аккаунта). Это сознательное упрощение MVP — см. раздел «Что дальше».

### Админ-панель (для имама)

Защищённая логином часть на `/admin` (вне языковых префиксов, интерфейс на русском).

**Один админ-аккаунт = одна мечеть.** У каждого `AdminUser` есть `mosqueId`; сессия (JWT) несёт этот `mosqueId`, и все списки/формы/действия в админке молча отфильтрованы по нему — имам мечети «Байкен» не видит хутбы и материалы мечети «Абу-Бакр», даже зная прямую ссылку на них (сервер отвечает 404). В боковой панели название своей мечети показано под логотипом. Ролей внутри одной мечети (главный имам/модератор) нет — см. «Что дальше».

| Страница | Маршрут | Описание |
|---|---|---|
| Вход | `/admin/login` | Email + пароль (NextAuth Credentials) |
| Обзор | `/admin` | Дашборд: количество хутб и материалов по разделам своей мечети |
| Хутбы — список | `/admin/khutbahs` | Таблица хутб своей мечети (черновики и опубликованные) |
| Хутбы — создание/редактирование | `/admin/khutbahs/new`, `/admin/khutbahs/[id]` | Форма с вкладками по языкам (kk/ru/en): заголовок, краткое описание, полный текст; дата, slug (URL, можно сформировать кнопкой из даты и заголовка), язык оригинала, статус публикации |
| Разделы сайта — список | `/admin/content` | Материалы своей мечети (сунны, рекомендации, напоминания, достоинства пятницы, Аль-Кахф) по вкладкам-категориям |
| Разделы сайта — создание/редактирование | `/admin/content/new`, `/admin/content/[id]` | Та же логика: вкладки по языкам, категория, порядок сортировки, статус публикации |

Всё через **Server Actions** (без отдельного REST API для CRUD), с серверной валидацией через Zod и понятными сообщениями об ошибках в форме.

**Заполнение на нескольких языках необязательно.** И для хутбы, и для материала раздела достаточно заполнить заголовок и текст **хотя бы на одном** языке — остальные локали автоматически покажут этот же текст с пометкой «перевод пока не готов» (см. `pickTranslation` выше), пока перевод не добавлен. В форме вкладки языков помечаются галочкой по мере заполнения, а счётчик под подсказкой показывает «заполнено N из 3».

### Модель данных (Prisma, `prisma/schema.prisma`)

```
Mosque               — мечеть (slug, name, order)
AdminUser            — имам/администратор (email, passwordHash), mosqueId — принадлежит одной мечети
Khutbah              — метаданные хутбы (slug, date, published, originalLocale), mosqueId — принадлежит одной мечети
  └ KhutbahTranslation — перевод хутбы (locale, title, summary, body) — по одной записи на язык
ContentBlock         — материал раздела (category: SUNNAH | RECOMMENDED_ACTION | REMINDER | FRIDAY_VIRTUE | AL_KAHF, order, published), mosqueId — принадлежит одной мечети
  └ ContentTranslation — перевод материала (locale, title, body)
QuranAyah            — один аят суры (surahNumber, ayahNumber, textArabic, transliteration) — общий для всех мечетей, без mosqueId
  └ QuranAyahTranslation — перевод аята (locale, text) — сейчас ru и en
```

Перевод на конкретном языке может отсутствовать — публичные страницы в этом случае автоматически показывают текст на языке оригинала (для хутб — язык оригинала хутбы, для аятов Корана — русский) с пометкой «перевод пока не готов» там, где применимо (см. `src/lib/i18n-content.ts:pickTranslation`). Для казахской локали перевод аятов Корана пока не найден в открытых источниках — казахоязычным читателям показывается русский перевод.

Текущая мечеть посетителя определяется в `src/lib/mosque.ts:getCurrentMosque` — по cookie `mosque` (слаг), а если её ещё нет, по первой мечети в порядке `order`. Смена мечети — server action `src/lib/mosque-actions.ts:setMosqueCookie`, вызываемая из клиентского переключателя в шапке.

**Данные Корана** (`prisma/data/al-kahf.json`) — арабский текст (издание `quran-uthmani`), английская транслитерация (`en.transliteration`), русский перевод Эльмира Кулиева (`ru.kuliev`) и английский перевод Saheeh International (`en.sahih`), полученные через публичный [alquran.cloud API](https://alquran.cloud/api) и один раз сохранённые в файл — во время `db:seed` сеть не требуется. Как и весь остальной контент, эти переводы стоит сверить с имамом перед боевым использованием.

## Установка и запуск

### Требования
- Node.js 20+
- npm

### Шаги

```bash
git clone https://github.com/dam1r-dev/JumaTime_V1.git
cd JumaTime_V1
npm install

# скопировать и заполнить переменные окружения
cp .env.example .env
# сгенерировать AUTH_SECRET и вставить в .env:
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"

# создать локальную БД по схеме
npx prisma migrate dev

# наполнить тестовыми данными (2 мечети, 2 админа, хутбы/материалы, аяты Аль-Кахф)
npm run db:seed

# запустить дев-сервер
npm run dev
```

Приложение будет доступно на `http://localhost:3000` (редиректит на `/kk`).

Сид создаёт **две мечети** («Байкен» и «Абу-Бакр») с раздельным контентом и по одному админу на каждую:

| Мечеть | Вход в админку | Логин / пароль (по умолчанию) |
|---|---|---|
| Байкен | `http://localhost:3000/admin/login` | `SEED_ADMIN_EMAIL` / `SEED_ADMIN_PASSWORD` в `.env` → `imam@jummatime.kz` / `ChangeMe123!` |
| Абу-Бакр | `http://localhost:3000/admin/login` | `SEED_ADMIN2_EMAIL` / `SEED_ADMIN2_PASSWORD` в `.env` → `imam2@jummatime.kz` / `ChangeMe123!` |

**Обязательно смените оба пароля** (или удалите второй демо-аккаунт) перед публичным использованием. Переключить, какую мечеть видно на публичных страницах, можно переключателем в шапке сайта.

### Полезные npm-скрипты

| Команда | Что делает |
|---|---|
| `npm run dev` | Дев-сервер (Turbopack) |
| `npm run build` / `npm run start` | Продакшен-сборка и запуск |
| `npm run lint` | ESLint по всему проекту |
| `npm run db:seed` | Пересоздать мечети, админ-аккаунты, тестовые хутбы/материалы и аяты Аль-Кахф (см. `prisma/seed.ts`) |
| `npx prisma studio` | GUI для просмотра/редактирования БД |
| `npx prisma migrate dev` | Применить/создать миграцию после изменения `schema.prisma` |

## Деплой

- **Фронт+бэк**: Vercel (проект — обычное Next.js-приложение, деплоится без доп. настроек).
- **База данных**: локальный файл SQLite не переживёт serverless-деплой Vercel (нет постоянной ФС). Перед деплоем нужно:
  1. Завести БД — например [Turso](https://turso.tech) (тот же libSQL-протокол, адаптер уже используется, менять код не нужно) или Postgres (Neon/Supabase) — тогда потребуется сменить провайдер в `prisma/schema.prisma` (`provider = "postgresql"`) и адаптер в `src/lib/prisma.ts`/`prisma/seed.ts` на `@prisma/adapter-pg`.
  2. Прописать боевой `DATABASE_URL` и `AUTH_SECRET` в переменных окружения хостинга.
  3. Прогнать `npx prisma migrate deploy` и один раз `npm run db:seed` (или создать первого админа вручную).

## Что дальше (для дальнейшей разработки)

Пункты не реализованы в MVP осознанно (чтобы не раздувать первую версию) — ближайшие кандидаты на доработку:

- **Учётные записи обычных пользователей.** Сейчас «Сохранённое» живёт в `localStorage` устройства — без синхронизации между устройствами. Полноценные аккаунты прихожан — следующий логичный шаг, если появится такая потребность.
- **Загрузка медиа.** Концепт упоминает возможные аудио/видео-версии хутб — сейчас поддерживается только текст.
- **Уведомления/пуши.** Раздел «Напоминания» сейчас статический контент, без реальных push-уведомлений о новых материалах.
- **Роли в админке.** Один `AdminUser` = одна мечеть, но внутри мечети роли не разделены (нет «главный имам» / «модератор» с разными правами).
- **Управление мечетями из UI.** Список мечетей сейчас редактируется только напрямую в БД (`prisma studio` или сид) — нет админ-экрана «создать мечеть + пригласить имама».
- **Английская/русская локализация редакторских уведомлений (toast) в админке** — сейчас интерфейс админки не локализован (это осознанно — рассчитан на одного оператора-имама).
- **Автотесты.** Пока только ручная QA-проверка в браузере, `tsc --noEmit` и `eslint`; unit/e2e-тестов нет.
- **Шариатская вычитка контента.** Все сид-данные (хутбы, материалы разделов) — иллюстративные примеры и должны быть заменены/проверены имамами перед реальным использованием (см. раздел «Вопросы шариатского характера» в исходном концепте). Текст Корана на странице Аль-Кахф взят из признанных изданий (Кулиев, Saheeh International), но перевод на казахский язык пока отсутствует в открытых источниках — при появлении официального перевода стоит его добавить.
- **Другие суры.** Модель `QuranAyah` уже поддерживает произвольный `surahNumber`, но сид и UI сейчас рассчитаны только на суру Аль-Кахф (18) — расширение на другие суры потребует доработки страниц.
